[BASE MODEL]

- **QoS Level 0:**
   - Ini adalah level QoS terendah di mana pesan akan diteruskan tanpa memeriksa apakah pesan tersebut sudah sampai atau tidak.
   - Jika server broker tidak bisa memastikan bahwa pesan telah diterima oleh client, maka pesan tersebut akan diabaikan atau dibuang.
   - Kelemahan dari QoS Level 0 adalah kemungkinan untuk hilangnya pesan jika ada gangguan pada koneksi.

- **QoS Level 1:**
   - Pada QoS Level 1, pesan dikirim satu kali dan setelah itu ditandai sebagai "terkirim".

Q: Jelaskan perbedaan antara HTTP dan MQTT untuk aplikasi IoT.
------------------------------------------------------------
[BASE MODEL]
HTTP (Hypertext Transfer Protocol) dan MQTT (Message Queuing Telemetry Transport) keduanya digunakan dalam pengembangan Aplikasi Internet of Things (IoT), tetapi mereka memiliki beberapa perbedaan penting:

1. **Prinsip Kerja**:
   - **HTTP**: HTTP adalah protokol data transfer web standar, dirancang untuk mengirimkan data dari server ke client (browser). Penggunaannya biasanya melibatkan permintaan-jawaban singkat. 
   - **MQTT**: MQTT adalah protokol publikasi-penerimaan, dimana pesan dikirim secara terus-menerus dari satu pihak (publisher) ke pihak lainnya (subscriber). Ini cocok untuk situasi di mana tidak semua pesan harus diterima segera.

2. **Kecepatan dan Kapasitas Data**:
   - **HTTP**: Ideal untuk interaksi real-time atau komunikasi yang memerlukan lebih banyak data, seperti streaming video atau audio. HTTP bisa sangat lambat ketika data besar diperoleh secara berurutan.
   - **MQTT**: Dirancang untuk mengirim pesan kecil dengan cepat dan efisien. Dapat menangani ribuan konsumen pada saat bersamaan, membuatnya ideal untuk IoT yang mungkin memiliki banyak sensor dan pelacak.
